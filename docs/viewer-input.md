# Viewer Input（入力の使い方）

## 概要

このセクションでは、decision-engine-coreに入力する値を設定します。

エンジンは：

入力（Input） → 定義（Definition） → 結果（Result）

の順で動作します。

---

## 1. 入力サンプル（Sample Input）

あらかじめ用意された「テスト用の入力例」を選択できます。

例：

- 通常状態になる想定の入力
- 温度上昇中の入力
- 高温状態の入力

これらは実際のセンサー入力を想定したサンプルであり、
状態そのものではなく「状態をテストするための入力」です。

---

## 2. 入力値の編集

以下の値を直接変更できます：

- value（現在値）
- previousValue（前回値）
- tempDelta（変化量）
- tempRate（変化率）
- tempRateAvg（平均変化率）
- previousState（前回の状態）

これにより：

👉 任意の状況を再現して、状態判定を確認できます

---

## 3. JSON表示（詳細）

内部では、これらの値はJSONとしてエンジンに渡されます。

通常はフォームで操作できますが、
必要に応じてJSONを確認することもできます。

---

## 4. 使い方の流れ

1. 入力サンプルを選ぶ
2. 必要に応じて値を調整する
3. Evaluateを実行する
4. 結果を確認する

---

## 5. 補足

この入力は「テストケース」として機能します。

将来的には、現場ごとの入力パターンを保存・共有することで、
制御ロジックの検証に活用できます。
